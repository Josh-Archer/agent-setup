# Codex Agents (repository layer)

This directory holds the **canonical Codex agent definitions** for this repository
(`*.agent.md`). Cross-provider architecture, role map, and delegation protocol live
in the repository root [`AGENTS.md`](../../AGENTS.md). Operational invocation details
are in [`docs/grok-agy-delegation.md`](../../docs/grok-agy-delegation.md).

Do not treat this file as a second source of truth for paths or role names. Prefer
the root `AGENTS.md` table and the frontmatter on each `*.agent.md` file.

## Layered instructions

| Layer | Location | Purpose |
| :--- | :--- | :--- |
| Cross-provider contract | Root `AGENTS.md` | Role map, sync workflow, delegation, `.agent-runs` handoffs, safety |
| Codex role definitions | `.codex/agents/*.agent.md` | Canonical role body + Codex model/tools frontmatter |
| Codex index (this file) | `.codex/agents/AGENTS.md` | How Codex agents relate to generated surfaces |
| Generated Grok surfaces | `.grok/agents/`, `.grok/roles/` | Produced by `scripts/sync_agent_surfaces.py` |
| Generated Antigravity plugin | `.agents/plugins/home-codex-agents/` | Produced by the same sync script |

Edit only `.codex/agents/*.agent.md` (and hand-authored root/docs files). After role
edits, regenerate surfaces:

```bash
python3 scripts/sync_agent_surfaces.py
```

Or verify generated trees match the canonical agents:

```bash
python3 scripts/sync_agent_surfaces.py --check
```

## Grok `agents_md` injection

Every generated `.grok/agents/<role>.md` sets `agents_md: true`. When Grok Build
runs a named agent, it injects repository project-rule files (root `AGENTS.md` /
`Agents.md`, plus deeper directory rules when applicable) into the session along
with the role body. That is why root `AGENTS.md` should stay the shared contract
rather than duplicating long instructions into every role file.

Antigravity does not auto-load root `AGENTS.md`. The plugin rule
`rules/repo-agents.md` (generated by sync) summarizes the same contract for agy.

## External delegation

The `grok-agy-delegate` skill lives at `.codex/skills/grok-agy-delegate/`.

- Single-agent entry point:
  `.codex/skills/grok-agy-delegate/scripts/delegate.py`
- Plan orchestration entry point:
  `.codex/skills/grok-agy-delegate/scripts/orchestrate.py`
- Handoffs: `.agent-runs/<run-id>/` (gitignored)
- Plan schema: `.codex/skills/grok-agy-delegate/references/plan-format.md`
- Guide: `docs/grok-agy-delegation.md`

Use a named agent's configured equivalent model. Pass `--model` only for an
intentional override.
